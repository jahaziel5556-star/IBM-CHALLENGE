# MatchMind One Codex Working Rules

This repository treats the documentation set as the source of truth.

## Rules

- Do not skip documented phases
- Keep frontend, backend, and AI orchestration concerns separate
- Never hardcode secrets
- Prefer maintainable, accessible implementations
- Keep the event engine deterministic and traceable

## Build Direction

- Implement the broadcast experience as a premium product, not a dashboard
- Use the event engine specification to drive prompts, explanations, and overlay timing
- Use IBM watsonx.ai / Granite through backend adapters only
